## Post Text

The dirtiest secret in data analytics:

Most dashboards are built on dirty data.

I've seen it at every company I've worked with:
→ Duplicate customer records nobody cleaned
→ Nulls treated as zeros in revenue reports
→ Date fields stored as text strings
→ "Updated" tables that overwrite history

And everyone just... knows.
But nobody fixes it.

Because fixing data quality is unglamorous.
No one gives you a promotion for it.
But bad data quality costs companies millions.

What I do now before building any dashboard:

1. Profile the data (row counts, nulls, duplicates)
2. Define what "clean" means for THIS use case
3. Document every transformation and assumption
4. Build in automated data tests (dbt tests are 🙌)

The best analysts I know are obsessed with data quality.
Not because it's exciting.
Because it's what makes everything else trustworthy.

What's the worst data quality issue you've ever found? 👇
